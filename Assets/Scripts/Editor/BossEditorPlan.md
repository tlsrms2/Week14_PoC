# 데이터 주도형(Data-Driven) 보스 패턴 에디터 시스템 설계 사양서

## 1. 개요 및 개발 목적
본 시스템은 기획자가 개발자의 스크립트 수정 및 개입 없이 **보스의 행동 패턴, 페이즈 전환, 세부 밸런스 수치를 직관적으로 조립하고 실시간으로 테스트**할 수 있는 환경을 구축하는 것을 목적으로 합니다. 데이터와 로직을 완벽히 분리하여 보스 콘텐츠 제작 생산성을 극대화하고, 유연한 확장성을 제공합니다.

---

## 2. 핵심 아키텍처 및 구현 방향

### 2.1 데이터 구조 (Data Structure)
* **ScriptableObject(SO) 기반 분리**: 보스의 능력치(Status), 현재 상태(State), 패턴 시퀀스(Sequence) 데이터를 C# 로직 코드와 완전히 분리합니다.
* 모든 수치와 연출 흐름은 SO에 데이터로 저장되며, 런타임에 보스 AI 컨트롤러가 이 데이터를 읽어 동작합니다.

### 2.2 UI 구현 방식 (Editor UI UI)
* **단계적 에디터 확장**:
  * **1단계 (초기/기반)**: 개발 비용 최소화를 위해 `ReorderableList`를 활용한 인스펙터 확장 방식으로 데이터 직렬화 및 순서 편집 기능을 먼저 도입합니다.
  * **2단계 (최종/고도화)**: `GraphView API`와 `UI Toolkit`을 활용하여 노드 기반의 비주얼 커스텀 에디터 윈도우(Custom Editor Window)로 전환하여 시각적 직관성을 완성합니다.

### 2.3 패턴 실행 로직 (Sequencer Architecture)
* **시퀀서 구조 및 커맨드 패턴**: 코루틴(Coroutine) 하드코딩 방식을 탈피합니다.
* 보스의 행동을 독립적인 '액션 블록' 단위로 설계하고, 시퀀서가 이 블록들을 순차적으로 읽어 실행하는 구조를 채택합니다.

---

## 3. 세부 기능 요구사항

### A. 모듈형 행동 블록 시스템 (Action Block System)
보스의 모든 행동을 단위별로 잘게 쪼개어 모듈화된 클래스로 제공합니다. 모든 블록은 공통 기저 클래스(예: `BossAction`)를 상속받아 구현됩니다.

* **이동 블록 (`MoveAction`)**: 플레이어 실시간 추적 타깃팅, 지정 좌표 이동, 특정 방향 대쉬(Dash) 등.
* **대기 블록 (`WaitAction`)**: 지정된 시간(초) 동안 대기하며 다음 행동 지연.
* **애니메이션 블록 (`PlayAnimAction`)**: 특정 애니메이션 상태 또는 클립(Clip) 재생.
* **탄막 발사 블록 (`FireProjectileAction`)**: 원형 형태, 부채꼴 스윕(Sweep), 샷건 등 다양한 투사체 발사 로직 처리.

### B. 시각적 패턴 시퀀서 (Visual Sequencer)
모듈화된 행동 블록들을 레고처럼 자유롭게 조립하여 하나의 거대한 '공격 패턴'을 구성하는 시스템입니다.

* **드래그 앤 드롭 편집**: 리스트 또는 노드 내에서 마우스 드래그를 통해 행동의 실행 순서를 자유롭게 변경할 수 있어야 합니다.
* **패턴 재사용성**: 조립된 하나의 패턴은 `AttackSequence` 구조의 독립된 SO 데이터로 저장되어, 다른 보스 캐릭터나 상이한 페이즈에서도 제한 없이 재사용할 수 있습니다.

### C. 조건 기반 의사결정 그래프 (Decision Graph)
보스가 언제 어떤 패턴을 사용할지, 다음 페이즈로 언제 전환할지 결정하는 의사결정 시스템을 시각화합니다.

* **상태 노드 (State Node)**: `Idle`, `Phase 1`, `Phase 2`, `Stagger (그로기)` 등 보스의 거대한 상태 상태를 정의합니다.
* **트랜지션 엣지 (Transition Line)**: 노드와 노드를 연결하는 선으로, 전환 조건을 내포합니다.
  * *조건부 예시*: 플레이어와의 사거리(Distance), 남은 체력/목숨 비율(HP Ratio), 특정 강력 패턴의 쿨타임(Cooldown) 완료 여부 등.
* **가중치 시스템 (Utility Weight)**: 동일한 우선순위나 조건이 충족되었을 때, 지정된 가중치에 따라 확률적으로 다음 패턴을 다이내믹하게 선택합니다.

### D. 애니메이션 완벽 동기화 (Animation Event Bridge)
스크립트의 단순 시간(초) 기반 타이머 방식의 오차를 해결하고, 애니메이션 프레임과 실제 공격 판정 시점을 완벽하게 일치시킵니다.

* **이벤트 브릿지 (`Animation Event Bridge`)**: 유니티 Animation 창에서 기획자가 직접 배치한 이벤트 마커(Animation Event) 신호를 시퀀서 에디터가 수신할 수 있도록 매핑 스크립트를 구성합니다.
* **마커 기반 트리거**: 특정 키프레임 마커에 도달하는 순간 탄막이 발사되거나, 근접 패링용 히트박스(Collider)가 정확 활성화되도록 연동합니다.

### E. 고유 기믹 확장을 위한 탈출구 (Extensibility - Escape Hatch)
기본 컴포넌트 블록만으로 구현이 불가능한 보스 고유의 '특수 기믹'을 프로그래머가 언제든 연결할 수 있도록 확장 통로를 제공합니다.

* **커스텀 이벤트 블록**: 에디터 내에 문자열(String ID)을 입력하면, 보스 오브젝트에 부착된 특수 하드코딩 스크립트의 특정 함수를 리플렉션 또는 메시징으로 호출합니다. (예: 지형 변화 트리거, 카메라 셰이크 연출).
* **프리팹 소환 블록**: 로직이 극도로 복잡한 독립적 기믹(예: 추적 레이저 망, 맵 배치형 장애물)은 에디터 내부에서 세부 수치를 제어하지 않고, 특정 타이밍에 해당 프리팹을 맵에 스폰(Spawn)하는 역할만 위임합니다.

### F. 시각적 피드백 및 실시간 밸런싱 (Visual Debugging & Hot Reload)
기획자의 작업 효율 및 실시간 밸런스 튜닝 생산성을 극대화하기 위한 편의 기능입니다.

* **핫 리로드 (Hot Reload)**: 유니티 플레이 모드(Play Mode)가 실행 중인 상태에서 에디터 상의 탄환 개수, 속도, 데미지, 쿨타임 수치를 변경하면 유니티를 껐다 켤 필요 없이 게임 내에 즉시 실시간 반영됩니다.
* **기즈모 및 핸들 연동 (`Gizmo & Handles`)**: 인스펙터/에디터 윈도우에 입력한 부채꼴 공격 각도, 플레이어 인식 범위, 사거리 등의 수치를 Scene 뷰에 직관적인 선과 도형(Gizmo)으로 실시간 드로잉합니다. 또한 Scene 뷰 내에서 마우스 핸들 조작을 통해 직관적으로 수치를 편집할 수 있게 지원합니다.
